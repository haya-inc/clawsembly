Commercial browser runtime

            
Local stays non-negotiable.

          

          

            The remote-sandbox route is rejected as the product default. The migration
            keeps execution in this tab, separates runtime vendors behind one boundary,
            and publishes every license and compatibility gate.
          

        

        

          
            01 / target
            
BrowserPod 2.xNode 22 · browser-local

            
Commercial Pro terms are explicit. Fastest migration candidate; metered API key, proprietary CDN runtime, and OpenClaw proof still required.

            Commercial
            Candidate ↗
          
          
            02 / escape hatch
            
container2wasmamd64 Linux · browser-local

            
Self-distributable feasibility lane. Experimental today; size, cold start, persistence, networking, SBOM, and guest-license duties remain gates.

            Apache-2.0 + guest
            Experiment ↗
          
          
            03 / baseline
            
WebContainerCurrent Chromium evidence

            
The verified Gateway slice remains a regression baseline. It is not the selected commercial production runtime and gains no new coupling.

            Evidence only ↗

            Rejected

Design principles

FIRST

        

          

            01 / USE
            
The official runtime.

            
OpenClaw remains the source of agent behavior, sessions, providers, and protocol.

          

          

            02 / DENY
            
Unknown capabilities.

            
Browser limitations fail closed with actionable diagnostics instead of dummy success.

          

          

            03 / SHARE
            
Every compatibility finding.

            
Reports, manifests, fixtures, and narrow upstream fixes are public OSS artifacts.
